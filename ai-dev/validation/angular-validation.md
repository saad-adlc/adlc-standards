# Angular validation guidance

## Grounded checks
- File names use hyphens.
- Unit test files use `.spec.ts`.
- Component TypeScript, template, and styles share the same base file name.
- Angular UI code lives under `src`.
- Feature-area grouping is used instead of type-based top-level folders.
- `inject()` is preferred over constructor parameter injection.
- Components stay focused on presentation.
- Templates avoid overly complex logic.
- `class` / `style` bindings are preferred over `ngClass` / `ngStyle`.
